Comparison of Python Web-frameworks

Overview

Pyramid

Start Small, Finish Big, Stay Finished

  • not a good choice for beginners
  • oriented towards large applications -> gets complicated quite quickly
  • documentation: cumbersome to navigate but tutorials explain every line of examples in detail
  • supported template engines: mainly Mako, Chameleon, Jinja2

Hello Pyramid

from wsgiref.simple_server import make_server
from pyramid.config import Configurator
from pyramid.response import Response


def hello_world(request):
    return Response('Hello Pyramid!')


if __name__ == '__main__':
    with Configurator() as config:
        config.add_route('hello', '/')
        config.add_view(hello_world, route_name='hello')
        app = config.make_wsgi_app()
    server = make_server('0.0.0.0', 6543, app)
    server.serve_forever()

Bottle

  • no dependencies (except Standard Lib) -> small size, hence convenient for Docker/Cloud deployments with few resources
  • appearance quite similar to Flask
  • easy to start with, even easier if you know Flask
  • clear and appropriate documentation
  • supported template engines: Cheetah, Mako, Jinja2, SimpleTemplate (inbuilt)

Hello Bottle

from bottle import route, run, template

@route('/hello')
def index():
    return template('<b>Hello Bottle</b>!')

run(host='localhost', port=8080)

Flask

web development, one drop at a time

  • the classic, widespread
  • supported template engines: primarily Jinja2, other possible
  • easy to start with, easy to extend
  • apt for prototypes as well as apps in production

Hello Flask

from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, Flask!'
